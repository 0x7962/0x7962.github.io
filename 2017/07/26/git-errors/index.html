git 错误汇总

2017-07-26

push失败

这是由于远程的git仓库不是用 git init –bare初始化所导致的，push的时候必须保证远程分支当前不在push的远程分支上

repo sync错误

这个是由于远程仓库的project_a没有这个master分支，或者这个project_a还是个空的仓库，可以先clone下来，然后push的时候再新建一个master的远程分支，记得再manifest文件里面的revision也要是这个master
